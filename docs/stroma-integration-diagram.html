Stroma Integration Plan

Reliability layer for McGill Course Explorer LLM pipelines

stroma[langgraph,redis] wraps existing LangGraph workflows — no orchestration replacement

Current State
5 LangGraph Workflows, No Reliability Layer
Every node wraps its body in try/except and returns empty results on failure. No retries, no checkpointing, no cost tracking, no structured observability.
Workflow Node Chain LLM Calls Failure Risk
Retrieval keyword → semantic → program → graph → structured → fusion Haiku, Voyage API timeouts, malformed SQL
Synthesis context_pack → synthesize Sonnet Token limits, timeout
Ingestion extract → chunk → embed → store Voyage Long-running, partial loss
Ingest scrape → resolve → embed Voyage, Playwright Hours-long, network failures
Planner gather_context → plan_agent Agent SDK Expensive, unpredictable cost
Phase 1
LangGraph Adapter — Contract Validation
Add Pydantic schema validation at every node boundary in the Retrieval and Synthesis workflows. No logic changes — decorators + adapter wrap only.
Retrieval Pipeline
keyword → semantic → program → graph → structured → fusion
Synthesis Pipeline
context_pack → synthesize
Pydantic Contracts
Input/output schemas per node. ContractViolation raised on shape mismatch — classified TERMINAL (no retries).
contracts.py per workflow
LangGraphAdapter.wrap()
Discovers @stroma_langgraph_node decorators, intercepts calls, validates boundaries. Non-decorated nodes pass through.
graph.py — 3 lines added

Modified: nodes.py (add decorators), graph.py (wrap compiled graph)
Unchanged: node function bodies, graph topology, state TypedDicts, API routes
Phase 2
StromaRunner — Ingest Pipeline Checkpointing
Migrate the ingest workflow (scrape → resolve → embed) to StromaRunner with Redis checkpointing. On failure, resume from the last successful node instead of restarting from scratch.
scrape → resolve → embed checkpoint after each
Redis Checkpoint Store
State persisted after each successful node. Survives container restarts. TTL configurable (default: 1 hour).
RedisStore via settings.redis_url
Failure Classification
RateLimitError → RECOVERABLE (3 retries). AuthError → TERMINAL (stop). Timeout → RECOVERABLE (backoff).
anthropic_classifier()
Resume-from-failure
If embed fails after 45 min of scraping: RunConfig(run_id="same-id", resume_from="embed")
scrape + resolve skipped — outputs loaded from checkpoint store
What changes
New: services/stroma/ (runner factory, classifiers), ingest/stroma_graph.py
Modified: config.py (redis_url setting)
Unchanged: existing LangGraph ingest graph (kept as fallback)
Phase 3
Cost Tracking + Budget Guards — Planner Workflow
Cap token/cost/latency on the most expensive workflow. The planner runs Claude Agent SDK with file tools — unpredictable cost per run.
gather_context → plan_agent budget-tracked
ExecutionBudget
tokens: 100,000 per plan
cost_usd: $5.00 per plan
latency_ms: 120,000 (2 min)
BudgetExceeded → RECOVERABLE
Usage Reporting
Nodes return (output, input_tokens, output_tokens, model).
Built-in pricing for claude-sonnet-4-6, claude-haiku-4-5.
result.total_cost_usd surfaced in SSE stream.
per-node token accounting
What changes
Modified: planner/graph.py (budget-capped runner, usage tuples), chat.py (surface cost in SSE)
Unchanged: gather_context logic, plan_agent prompt/tools
Phase 4
Observability Hooks — Structured Tracing
Replace ad-hoc logger.warning() with structured lifecycle hooks. Three async callbacks: on_node_start, on_node_success, on_node_failure.
NodeHooks
on_node_start(run_id, node_id, input)
on_node_success(run_id, node_id, output, tokens)
on_node_failure(run_id, node_id, exc, class)
services/stroma/hooks.py
ExecutionTrace
Full audit trail per run. trace.failures() for errors.
trace.to_json() for log export.
trace_a.diff(trace_b) for regression detection.
available on every result
OpenTelemetry Ready
Swap hook callbacks with OTel span management when tracing infra is added. No pipeline code changes needed.

Migration Path

Phase Effort Risk Blast Radius Value
1 Adapter Low Minimal Retrieval + Synthesis Catch silent failures at boundaries
2 Checkpoints Medium Low Ingest pipeline only Resume-from-failure, auto retries
3 Budgets Low Minimal Planner workflow Cost control + visibility
4 Hooks Low None All runners Structured observability

Each phase is independently deployable. Recommended order: 1 → 2 → 3 → 4.
